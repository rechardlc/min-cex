# vehivle V1：展示型数字展厅（项目上下文技能）

## 触发与输出（本技能的用法）

当用户提到以下任一类内容时，优先调用本技能来“补上下文 + 对齐约束”再进入实现：
- 项目关键词：vehivle、两/三轮车、渠道分销、电子展厅、画册、小程序展厅、运营位
- 业务对象：车型、分类、参数模板、媒体上传、价格策略、免责声明、分享落地、下架兜底、RBAC、审计日志
- 技术对象：`/api/v1/*`、`draft/published/unpublished/deleted`、`msrp_price`、`upload-policy/complete`

默认输出顺序（除非用户要求直接写代码）：
1) V1 范围边界（做/不做）
2) 相关硬性规则（状态/口径/校验/兜底）
3) 需要确认的问题（如果有）
4) 可执行的实现清单（端到端：后端/后台/小程序/测试验收）

## 你在做什么（先对齐一句话）

这是一个**无需登录的纯展示型“数字化电子画册”**：下游门店/散客快速看车、查参数、联系转化；**V1 不做交易/支付/订单/客户体系**。

参考文档（以 `doc/` 为准）：
- `doc/需求文档_PRD.docx`：产品范围、页面、字段、规则、验收、上线检查、Roadmap
- `doc/tech.md`：技术栈/数据建模/API 摘要/性能与异常兜底
- `doc/接口详细版_请求响应错误码字段校验.docx`：接口明细、请求/响应示例、错误码、字段校验、分页/鉴权约定

## V1 范围边界（强制记忆）

**V1 必做**
- 小程序：首页（Banner/金刚区/运营专区/联系入口）、分类选车库（触底加载/下拉刷新）、车型详情（媒体/参数/价格口径/免责声明）、拨号/微信联系、原生分享
- 后台：账号密码登录、简化 RBAC、车型管理（草稿/上架/下架/删除）、分类管理（最多两级）、参数模板、系统设置、运营位配置、（可选）操作日志
- 口径：价格展示策略、免责声明、异常状态页、埋点预埋

**V1 不做**
- 下游客户登录/审核/分级、千人千面价格
- 购物车/订单/支付/库存同步/ERP 对接
- IM/消息中心/直播/全站搜索/复杂报表

## 关键域模型（术语/表/字段要一致）

### 状态机（必须统一）
- PRD：草稿 → 已上架 → 已下架 → 逻辑删除
- 接口/校验通则：`draft / published / unpublished / deleted`

约束：
- **仅 `published` 对小程序公开可见**；`draft/unpublished/deleted` 必须走兜底（尤其旧分享链接）。

### 核心实体（V1）
- `vehicles`：车型内容（名称/分类/媒体/参数/卖点/价格/标签/排序/状态/分享信息）
- `categories`：分类（最多两级；一级用于大类，二级用于品牌/系列，可选）
- `param_templates` + `param_template_items`：参数模板（按一级分类绑定；项支持 文本/数值/单选；必填/前台展示/排序）
- `media_assets`：媒体元数据（对象存储直传后回写生成）
- `system_settings`：全局设置（公司名、电话/微信/二维码、默认分享图、默认价格模式、免责声明等）
- `audit_logs`：关键操作审计（发布/下架/删除/系统设置变更等）

## 硬性规则清单（实现时逐条自查）

### API/数据一致性
- **对外 ID 一律 `string`**（请求入参也用 string；禁止 number）
- 时间统一 **ISO8601**（带时区，例如 `2026-03-15T10:20:30+08:00`）
- 统一响应结构：`code/message/data/request_id/timestamp`
- 分页：`page>=1`，`page_size` 1~100（注意 PRD 的列表首屏/分页节奏要求）
- 排序：`sort_order` 整数（建议 0~9999），**值越大越靠前**；同值按 `updated_at` 倒序

### 安全/权限（后台）
- 后台接口 `Authorization: Bearer {access_token}` 必填
- 简化 RBAC：**超级管理员**（含账号/系统设置/日志） vs **运营编辑**（内容维护与运营位）

### 价格口径（前后台必须一致）
- 全局策略：`显示建议零售价` vs `电话询价`
- 若全局为显示价格，但某车型未填 `msrp_price`：**必须回落为“电话询价”**
- V1 仅允许 `msrp_price`（不得出现底价/成交价等交易口径）

### 媒体上传（后台操作体验/约束）
- 图片建议 ≤ 2MB；视频 ≤ 20MB
- 图片超限：**自动压缩后再上传**，压缩后仍超限则阻断并提示
- 视频超限：**直接阻断**（V1 不做自动压缩）
- 富文本（卖点等）必须做 **XSS 过滤**
- URL 仅允许 **https**，且对象存储域名需白名单

### 小程序兜底/异常（避免白屏）
- 无 Banner：模块隐藏
- 分类无车型：空状态文案（建议“当前分类暂无车型...”）
- 图片加载失败：占位图，其他内容可正常看/联系
- 详情无参数：不展示空表，显示统一提示语
- 未配置电话/微信：隐藏对应按钮（不留空白占位）
- 下架/删除车型旧链接：下架兜底页（返回首页 + 联系上游）
- 接口异常/弱网：加载失败提示 + 重试按钮

## 接口契约（摘要版，用于实现/联调不跑偏）

### 管理端（admin）
- 登录：`POST /api/v1/admin/auth/login`
- 刷新：`POST /api/v1/admin/auth/refresh`
- 当前用户：`GET /api/v1/admin/auth/me`
- 改密：`POST /api/v1/admin/auth/change-password`
- 分类 CRUD：`/api/v1/admin/categories`
- 参数模板：`/api/v1/admin/templates`（注意：路径在接口文档中给出，以其为准）
- 车型列表/创建/更新/详情：`/api/v1/admin/vehicles`
- 媒体直传：`POST /api/v1/admin/media/upload-policy` → 直传 → `POST /api/v1/admin/media/complete`

### 公开端（public，小程序）
- 首页聚合：`GET /api/v1/public/home`（仅返回 `published`）
- 分类树：`GET /api/v1/public/categories`
- 分类车型列表：`GET /api/v1/public/vehicles`（分页）
- 车型详情：`GET /api/v1/public/vehicles/{id}`（仅 `published`）
- 分享落地校验：`GET /api/v1/public/vehicles/{id}/share-check`（未上架/已下架/删除返回兜底文案）
- 联系配置：`GET /api/v1/public/contact`

## UI/UX & 埋点 & 合规（经常被遗忘的点）

### 性能指标（验收口径）
- 首页首屏 ≤ 3 秒；列表/详情核心内容 ≤ 2 秒
- 支持长期维护 300 条车型数据

### 埋点（V1 先预埋）
- 优先使用**微信小程序原生自定义分析**，不额外自建埋点服务
- 事件：`home_view`、`banner_click`、`category_click`、`zone_click`、`vehicle_view`、`phone_click`、`wechat_click`、`share_click`

### 合规文案（避免提审风险）
- 前台只用“查看详情/立即询价/联系上游”等展示型文案
- **禁止**出现“购买/下单/支付/加入购物车”等交易字样
- 默认免责声明建议（价格/参数/图片）需可后台配置，未配置用默认文案兜底

## 易踩坑提醒（实现前先看一眼）

- **公开接口只出 `published`**；后台预览必须依赖 `preview_token`（接口文档有该约定）。
- 发布后若有 CDN：发布成功要考虑**清理/刷新缓存**（tech/接口文档提到发布清缓存诉求时要落到实现方案）。
- “PRD 页面约束”与“接口通则”可能有冲突需要定稿：
  - PRD：详情页媒体最多 **9 张图**；接口通则：图片数组最多 **20**（建议以 PRD 为准，或明确后台可存 20 但前台只取前 9）。
  - PRD：列表首屏/分页建议 **10**；接口默认 `page_size=20`（建议统一联调口径）。
- 分类停用后的挂载车型如何处理（迁移/隐藏）要在实现里体现，不要放任脏数据导致前台路径异常。
